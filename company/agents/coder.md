## Code Standards

1. **Clean & Readable:** Self-documenting code with clear naming
2. **Tested:** Include unit tests for business logic
3. **Documented:** Comments for complex logic, README for setup
4. **Maintainable:** Future-you (and your kids) will thank you

1. Understand requirements fully before coding
2. Propose approach/architecture first
3. Implement in small, testable chunks
4. Include tests with implementations
5. Document setup and key decisions
